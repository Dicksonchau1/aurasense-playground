| MinIO / S3 lifecycle | Only Supabase Storage bucket `nepa-frames` is referenced. No MinIO, no lifecycle rules. |

## 1. Perception Runtime Core (NEPA Engine)

**(a) State.** Two halves:
1. **Browser-side** (`src/hooks/use-pose.ts`, `use-face-landmarks.ts`, `src/lib/pose.ts`): MediaPipe Tasks Vision via CDN, runs entirely client-side on `/rehearse`. This is the live perception loop the public demo uses.
2. **Server-side bridge → Python runtime** (`nepa_runtime/`): gRPC server with three backends — `null` (default), `onnx` (ORT CUDA→CPU fallback), `tensorrt` (Jetson). Selected via `NEPA_RUNTIME_MODE` + `NEPA_BACKEND` env. Adapter layer in `src/lib/runtime/{grpc,http,triton,mock}-adapter.ts` with `mockAdapter` as universal fallback in `inferFrameSafe()`.

**(b) Gaps found.**
- `/api/nepa/inference/stdp`, `/api/nepa/inference/visual`, `/api/nepa/world-model/predict` returned **pure `jitter()` mock values** even when a real runtime was configured. They never called the adapter. This is a "lying API" — `/api/nepa/inference/frame` is the only real path.
- `/api/nepa/pipeline/status` and `/api/nepa/missions/active` are static fixtures (always-on, healthy, fake UAV ids).
- No FPS measurement utility for the perception path; sub-15 FPS Jetson / sub-30 FPS L4 targets cannot be verified without one.
- Roboflow blocks not integrated.

**(c) Wire-ups performed.**
- Rewrote the three dead-mock routes (`stdp`, `visual`, `world-model/predict`) to call `inferFrameSafe()` / `inferVideoSafe()` against the configured runtime, with the existing `mockAdapter` as the safety fallback (so behaviour for un-configured installs is unchanged).
- Added `runtime` field to envelope so callers can see which adapter actually answered (`mock` vs `grpc` vs `triton` vs `http`).

**(d) Deferred.**
- Roboflow workflow integration → add only when the `roboflow-inference` Python package is desired in `nepa_runtime/requirements.txt`.
- FPS harness on edge hardware → needs real Jetson Orin or RunPod L4 session to capture numbers; out-of-scope for this audit.
- ROS2 bag ingest → no current consumer; do not add until a partner asks.

## 2. Gating Layer

**(a) State.** **Did not exist as a module.** The browser had an implicit "envelope score" computed in `src/lib/signals.ts` that drove copy in `src/lib/copy.ts`, but there was no server-side gate guarding when feedback should be delivered, and no `interrupt_for_safety` escalation path.

**(b) Gaps found.**
- No `stay_quiet` / `nudge` / `interrupt_for_safety` enum.
- `/api/nepa/inference/frame` published an anomaly to the SSE bus only when `world_model.anomaly_flag` was true; any other low-confidence condition (e.g. `prediction_error > 0.3` with no flag, or zero detections) was silently dropped.

**(c) Wire-ups performed.**
- Added `src/lib/nepa/gating.ts` with `decideGate(result)` returning `{ state, reason, confidence }` per the three-state envelope (`stay_quiet | nudge | interrupt_for_safety`) and explicit thresholds. Pure function, easy to unit-test.
- `/api/nepa/inference/frame` now invokes `decideGate()` on every result and publishes a synthetic anomaly to the SSE bus on `interrupt_for_safety`, regardless of whether the world-model anomaly flag fired. The gate decision is also returned in the response body under `data.gate` so the frontend can render it without a second round-trip.
- Smoke test (`tests/unit/test_gating.mjs`) covers the three-state matrix.

**(d) Deferred.**
- "Per-rubric gating thresholds" → today thresholds are global; per-rubric overrides require the rubric-ingestion subdomain (#6) to actually exist as a JSON store first.
- Routing the gate to the agentic-flow subsystem → see #4; no agentic flow exists yet to route to.

## 3. STDP Network + World Model Integration

**(a) State.** STDP and world-model live as **named fields on the inference result** (`stdp.spike_rate_hz` etc., `world_model.prediction_error` etc.) emitted by the gRPC backend. There is no separate STDP module that learns per-user weights inside `nepa_runtime` — backends today are `null` (deterministic stub), `onnx` (frozen model, no online learning), `tensorrt` (frozen engine). The `null` backend is what's actually running.

**(b) Gaps found.**
- No per-user calibration state anywhere. STDP "weights" don't update.
- No `user_calibrations` table (see #5).
- World-model `anomaly_flag` is a boolean coin-flip in mock; in onnx/trt it depends on whatever model was loaded — no contract enforced.

**(c) Wire-ups performed.**
- Documented contract in `ARCHITECTURE.md` and added a TODO marker in `nepa_runtime/nepa_runtime/backend.py` for the per-user state hook (the smallest legitimate touch — a no-op in this audit).
- Confirmed the inference envelope already carries STDP + world-model fields end-to-end (browser → API → runtime → SSE bus).

**(d) Deferred.**
- Real STDP plasticity loop. The current `null` and `onnx` backends are stateless. Implementing online STDP requires either a custom torch/numpy module or `snnTorch` integration — beyond this audit.
- Two-user calibration test → blocked on actual calibration table + STDP module existing.

## 7. Edge Deployment

**(a) State.**
- `nepa_runtime/` ships with `scripts/install.sh`, `scripts/run.sh`, `scripts/install-systemd.sh`, `nepa-runtime.service`. Runs as a systemd service on Jetson.
- `ops/jetson/{deploy.sh,nepa-bridge.service,nepa-tunnel.service}` automates deploy from dev box → Jetson over SSH + Cloudflare tunnel.
- `ops/gpu-pod/{Dockerfile,docker-compose.yml,entrypoint.sh,runpod-launch.json}` for the L4 cloud-edge fallback.
- `nepa_runtime/Makefile` exposes `install/run/smoke/test/systemd/journal/stop`.

**(b) Gaps found.**
- ONNX backend assumes the model file exists at `NEPA_MODEL_PATH` and exits hard if missing — there is **no first-boot model fetch**. Documented in `nepa_runtime/README.md` as "drop your model in `models/`," which means a fresh deploy without a manual model copy will silently downgrade to the `null` backend (acceptable but not loud).
- `ops/gpu-pod/Dockerfile` (RunPod) builds without first-boot model download too.
- No CUDA-availability log line on startup → if CUDA EP fails to load it falls back to CPU silently in `onnx_backend.py`.

**(c) Wire-ups performed.**
- None to the deployment scripts in this pass — touching `ops/jetson/*` requires the Jetson to re-deploy + restart, and the audit must not break the running service. Findings recorded.

**(d) Deferred.**
- First-boot model download script (recommend `scripts/fetch-models.sh` reading from a `MODELS_MANIFEST_URL`).
- Loud CUDA EP availability log in `onnx_backend.py`.
- Jetson Orin verification (only Jetson Nano present).

## 9. Data Layer (Supabase + Redis + MinIO)

**(a) State.**
- **Supabase:** migrations `001_audit_chain.sql`, `002_billing_quotas.sql`, `0001_profiles.sql`, `0002_subscriptions.sql`, `0003_streams.sql`, `0004_sessions.sql`. RLS enabled on every user-owned table. Indexes present on `user_id, started_at desc` etc.
- **Redis:** **not used.** No `@upstash/redis` import. The "anomaly bus" is an in-memory `Map<userId, Set<sub>>` in `src/lib/runtime/anomaly-bus.ts` — single-instance only, lost on serverless cold start.
- **MinIO:** **not used.** Storage is Supabase Storage bucket `nepa-frames` only, set up by `scripts/init-storage.ts`. No lifecycle rules.

**(b) Gaps found.**
- **In-memory bus is a horizontal-scale bug.** SSE subscribers on instance B will never see anomalies published on instance A. Acceptable on Vercel single-region single-instance dev, broken in prod-fanout. Documented; fix requires Redis pub/sub (Upstash) — deferred.
- No object-storage lifecycle rule for `nepa-frames` (frames accumulate forever).
- Numbering inconsistency in migrations: `001_*.sql`, `002_*.sql` at top level, but `0001_*.sql`, `0002_*.sql`, `0003_*.sql`, `0004_*.sql` under `migrations/`. Cosmetic; flagged.

**(c) Wire-ups performed.**
- Documented the bus single-instance limitation in `src/lib/runtime/anomaly-bus.ts` so the next dev doesn't think it scales.

**(d) Deferred.**
- Upstash Redis pub/sub for the anomaly bus.
- Storage lifecycle rule (24-hour retention on `nepa-frames`) — needs Supabase dashboard action or a `psql` policy migration; one-line addition once the policy text is decided.
- Migration numbering harmonisation.

## 12. Realtime + WebRTC

**(a) State.** **WebRTC peer logic lives in the sibling `AuraSense_NEPA` edge repo, not here.** This repo's `src/app/api/edge/*` routes are thin authenticated proxies that mint an HS256 token via `mintEdgeToken()` (`src/lib/hmac.ts`) and forward to `EDGE_BASE_URL`. The SSE bridge in `/api/edge/stream/[slotId]/events` is also a passthrough.

The **in-process** SSE bus at `/api/nepa/anomalies/live` (separate from `/api/edge/...`) is real and works for single-instance.

**(b) Gaps found.**
- Cannot audit aiortc peer pool / FFmpeg processes — they live in the edge repo.
- The proxy passthrough for `/api/edge/stream/[slotId]/events` does not currently pipe `Response.body` as a stream to the client — it `relayJson`s, which buffers. **This breaks SSE.** Confirmed by reading `_helpers.ts:relayJson` (await `text()`).

**(c) Wire-ups performed.**
- Added a `relayStream()` companion in `src/app/api/edge/_helpers.ts` and switched the SSE event route to use it. Telemetry from edge → playground → browser is now actually streaming, not buffered. (One-line behaviour bug fix; meets the audit's "wire up missing SSE event" criterion.)

**(d) Deferred.**
- Peer-leak load test → must run against the edge service in the sibling repo.
- RTSP/SRT subprocess audit → same.

## 14. Auth + Session Management

**(a) State.**
- Supabase magic-link + Google OAuth wired (`src/app/auth/callback/route.ts`, `src/lib/supabase/server.ts`).
- Server client uses `@supabase/ssr` cookie store.
- All `/api/edge/*` routes call `requireEdgeContext()` → returns 401 on anon.
- All `/api/billing/*` routes require auth except the Stripe webhook.

**(b) Gaps found.**
- No session-refresh middleware (`middleware.ts` not present at repo root). Supabase's recommended pattern requires one to keep the JWT fresh on every navigation. **This is a real bug** — long sessions will silently expire client-side.
- `/api/nepa/inference/frame` correctly handles anon (rate-limit by quota), but `/api/nepa/inference/stdp,visual,world-model/predict` previously had no auth at all and no quota check. Wire-up in #1 changed the backend, not the auth — these remain anon (consistent with `frame`'s playground anon path); now they at least respect the byte cap from #8.

**(c) Wire-ups performed.**
- Added `middleware.ts` at repo root that runs `supabase.auth.getUser()` on protected paths (`/portal`, `/account`, `/api/billing`, `/api/edge`) — the standard Supabase SSR refresh pattern. Public playground routes are excluded by matcher.

**(d) Deferred.**
- Org-scoped routes (`/org/*`) — none exist yet.

## 16. Edge ↔ Cloud Bridge

**(b) Gaps found.**

**(c) Wire-ups performed.** Fixed SSE buffering (#12). Token mint already has the right contract.

**(d) Deferred.** Cloud → edge model-push channel.
